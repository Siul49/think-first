# Code Reading - Execution Protocol

## Step 0: 레벨 결정

1. 기본값: **레벨 C** (응용 수준)
2. 사용자가 "바빠", "급해", "빨리" 등을 명시 → 레벨 B 또는 A
3. 레벨 전환 시 사용자에게 확인

## Step 1: 변경 범위 파악

1. `git diff` 또는 작업 내역으로 변경된 파일/함수 목록 확인
2. 핵심 변경과 부수 변경 구분
3. 질문 대상 = **핵심 변경** (설정 파일, 포맷팅 등은 제외)

## Step 2: 질문 순서 결정

1. 진입점(entry point)부터 시작 — Router → Service → Repository 순
2. 새로 생성된 코드 우선, 수정된 코드 후순위
3. 복잡한 로직이 있는 부분에 더 많은 질문 배정

## Step 3: 질문 실행

### 레벨 C (기본)

파일/함수별로 3단계 질문:

```
📖 [파일명:라인]

1️⃣ What: 이 코드가 뭘 하는지 설명해봐
2️⃣ Why: 왜 이렇게 구현했어? 다른 방법은 없었을까?
3️⃣ 응용: 여기에 {구체적 시나리오}를 추가한다면 어디를 수정할 거야?
```

### 레벨 B

```
📖 [파일명:라인]

1️⃣ What: 이 코드가 뭘 하는지 설명해봐
2️⃣ Why: 이 방식의 장단점은 뭘까?
```

### 레벨 A

```
📖 [파일명:라인]

1️⃣ 이 코드의 실행 흐름을 순서대로 설명해봐
```

## Step 4: 답변 평가 및 후속

| 답변 | 대응 |
|------|------|
| 정확 | "맞아요!" → 다음 질문 |
| 부분 이해 | "거의 다 맞는데, {빠진 부분}도 생각해봐" |
| 오해 | "그 부분이 좀 다른데 — {힌트}. 다시 생각해볼래?" |
| 모름 | 코드를 함께 읽으며 설명 → "그러면 이건 뭘 의미할까?" 확인 질문 |

## Step 5: 완료 판단

- 모든 핵심 변경에 대해 레벨 기준 질문 통과
- "모름" 답변은 설명 후 이해 확인까지 완료
- 사용자가 "충분해", "넘어가자"로 명시적 종료 가능

완료 시:
```
✅ 코드 리딩 완료! {통과한 항목 수}/{전체 항목 수}
{간단한 피드백 — 잘한 부분, 보완할 부분}
```

## On Error

- 사용자가 반복적으로 같은 개념을 모르면: 해당 개념에 대한 학습 자료 방향 제시
- 컨텍스트 부족으로 질문이 어려우면: 핵심 변경 1-2개에만 집중
