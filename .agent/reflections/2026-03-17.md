  - **bash 정규식 이스케이프**: 싱글쿼트/더블쿼트에서 `\.` vs `\\.` 차이 등
- early return 패턴 vs 플래그 패턴: 두 관심사를 한 스크립트에 넣을 때 early return이 후속 로직을 차단하는 문제를 이해함

- 훅/bash/exit code 등 기초 개념부터 자세한 설명을 듣는 시간을 별도로 갖기

## 작업: memory 시스템 이해 + 승격 기준 정립

### 배운 점
- **reflection과 memory의 차이**: reflection은 날짜별로 쌓이는 학습 기록(수동 조회), memory는 매 세션 자동 로드되는 인덱스. 역할이 다르다
- AI가 이전 세션의 맥락(기획 결정 등)을 아는 이유가 memory 자동 로드 덕분이라는 걸 이해함
- memory 승격 기준 3가지를 직접 정립: (1) 기획적 결정 + 이유 (2) 기술 선택의 why (3) Code Study에서 확실히 이해한 것
- 공통 원칙: 코드에서 읽을 수 있는 what은 저장 불필요, 판단의 why만 저장

### 다음에 다르게 할 점
- 회고마다 "이건 memory에 올릴 만한가?" 판단을 습관화
- reflection을 취업 준비/학습 교재로 활용 — 단순 기록이 아니라 다시 펼쳐볼 자산으로 관리

### 예상과 달랐던 점
- memory가 단순 저장소가 아니라 "자동 로드되는 인덱스"라는 점이 핵심 차이였음. 파일이 있다고 다 쓰이는 게 아니라, MEMORY.md에 등록해야 다음 세션에서 인식됨
