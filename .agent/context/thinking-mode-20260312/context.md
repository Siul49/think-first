# Thinking Mode — 컨텍스트

## 배경

경수님이 skill-pack을 개인 성장 도구로 전환하고자 함.
"AI가 대신 해주는" 패턴에서 "AI가 사고를 강제하는" 패턴으로 전환.
배포 가능성은 유지 — 이 컨셉 자체가 프로젝트의 차별성.

## 핵심 결정사항

| 결정 | 선택 | 근거 |
|------|------|------|
| 강도 | 모든 작업에 동일 사이클 | 질문 깊이는 작업 복잡도에 자연 비례 |
| 회고 저장 | `.agent/reflections/YYYY-MM-DD.md` | 날짜별 파일로 성장 기록 축적 |
| 스킵 정책 | 불가 (최소 1개 질문 응답 필수) | 엄격한 사고 강제가 목적 |
| 배포 성격 | 유지 (기본값이 사고 강제 모드) | 차별성 확보 |

## 제약

- 검증 스킬(verify-*)과 서브에이전트는 변경하지 않음 (자동화 목적 유지)
- Hook 스크립트도 변경하지 않음 (시스템 레벨 자동화)
- 기존 스킬의 핵심 로직은 유지, 사고 사이클만 추가

## 리스크

| 리스크 | 대응 |
|--------|------|
| 모든 작업에 질문이 부담될 수 있음 | 질문 깊이를 작업 복잡도에 비례하게 설계 |
| 회고 파일이 형식적으로 채워질 수 있음 | 구체적 질문으로 유도 (단순 Yes/No 불가) |
| 다른 사용자가 사고 강제를 원치 않을 수 있음 | 향후 모드 전환 고려 (현재 범위 밖) |
